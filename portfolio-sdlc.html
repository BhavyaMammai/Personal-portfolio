SDLC PROJECT - Documentation

The SIX Project was an initiative aimed at conceptualizing a revolutionary streaming device to enhance the TV-watching experience. This device, envisioned to be connected via HDMI and Wi-Fi, would allow users to stream content from popular platforms like Netflix and Amazon Prime directly to traditional TVs. It would also enable users to manage their streaming accounts, control the device remotely, and access detailed content information through platforms such as IMDb.

While the project was not fully developed, I was involved in creating the full Software Development Life Cycle (SDLC) plan and various technical documentation as part of the pre-development process. The goal was to simulate the planning, design, and deployment phases that would be followed in a real-world product development cycle.

My primary contribution to the SIX project involved laying out a comprehensive SDLC strategy. This included defining the project requirements, creating design documents, and formulating detailed strategies for development, testing, and deployment. The approach was intended to showcase how an organization would approach the development of a product of this scale, from initial concept to launch.

1. Requirements Gathering

In this phase, I collaborated with stakeholders and conceptualized the core features and functionalities of the SIX device. I focused on defining both functional and non-functional requirements to ensure the device would meet user needs and industry standards. This phase involved:

  • Defining core functionalities such as streaming content, account management, and remote control operation.
  • Identifying non-functional requirements, including performance, security, and scalability, to ensure the device would be reliable, secure, and able to handle large amounts of data and traffic.

2. System Design

The next step was to create system design documents. This involved conceptualizing how the device would interact with users and other systems. Some of the key design documents I created include:

  • System Diagrams: I created several diagrams to model the flow and interactions of the system, such as:
    • Context Diagram: To outline the system's scope and boundaries, illustrating how the SIX device interacts with external systems like streaming platforms, IMDb, and user devices.
    • Entity Relationship Diagram (ERD): To define the database structure, illustrating how user data, streaming subscriptions, and content metadata would be stored and managed.
    • Data Flow Diagram (DFD): To show the movement of data between the SIX device, users, and external services.

3. Development Planning

Although the actual development was not carried out, I developed a detailed development plan for the project. This phase focused on breaking down the core features of the device into smaller, manageable tasks. I:

  • Defined key features such as account registration, user subscription management, and seamless integration with streaming platforms.
  • Identified potential technical challenges (e.g., Wi-Fi integration, content streaming, and API interactions with third-party platforms).
  • Proposed tools, frameworks, and technologies that would be used in the actual development phase (e.g., cloud infrastructure for content management, REST APIs for communication, etc.).

4. Testing Strategy

Creating a comprehensive test plan was crucial in ensuring the product would meet its quality standards. I outlined the testing strategy that would be used to validate the features and performance of the device. This strategy included:

  • Unit Testing: To test individual components of the system (e.g., account management, subscription handling).
  • Integration Testing: To test how different parts of the system (e.g., the device interface, APIs, and the remote control) would work together.
  • System Testing: To ensure the overall system would function as expected under various scenarios, such as high user traffic, poor network conditions, or errors in external APIs.

5. Deployment Strategy

The final step of the planning phase involved outlining the deployment and rollout strategy. I developed a plan for:

  • System-level testing to ensure the platform could handle large-scale deployment and real-world usage.
  • Deployment Phases: I suggested an initial launch phase in Japan, with a step-by-step rollout plan to test the device and its functionality in a localized market before expanding globally.
  • Continuous Monitoring and Updates: Post-launch monitoring strategies were also defined to track performance, user feedback, and potential bugs to ensure timely updates and fixes.

Key Contributions

  • SDLC Documentation: I produced detailed documentation covering each phase of the SDLC. This includes requirements, system design, testing strategies, and deployment plans. While no actual code was developed, the documentation is a full simulation of how the project would proceed if developed.
  • System Design & Architecture: I created several essential design documents such as context diagrams, entity relationship diagrams (ERDs), and data flow diagrams (DFDs). These diagrams provided a conceptual understanding of how the SIX device would interact with users and third-party services.
  • Testing Strategy: I defined a structured test plan that would be followed to ensure the device met the defined requirements. This included unit testing, integration testing, and system-level testing.
  • Deployment Planning: I created a deployment strategy with phased implementation and post-launch support, ensuring that the system could be scaled globally after initial testing in Japan.

Conclusion

The SIX Project allowed me to demonstrate my ability to plan and structure a complex software development effort. While the project was never fully developed, the SDLC documents serve as a valuable example of how a real-world product development process might unfold. The planning and documentation process not only enhanced my skills in system design and project management but also deepened my understanding of how different SDLC phases interrelate.

The work completed in this project reflects my ability to:

  • Gather and analyze requirements
  • Create comprehensive design and architecture documentation
  • Develop detailed testing strategies
  • Plan for real-world deployment and maintenance

Through this project, I was able to simulate a real-world software development environment and gain valuable insights into how an organization would tackle a large-scale product launch in the competitive streaming industry.
